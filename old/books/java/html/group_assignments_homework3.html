3.1. FTP Viewer

This program should implement a graphic front for a FTP client, that is able to provide the same functionality the following commands offer:

  • ls - List the current directory in the server.

  • cd - Go to a directory in the server.

  • lls - List the current directory in the client.

  • lcd - Go to a directory in the client.

  • put - Put a file in the server current directory.

  • mput - Put all files that match a wildcard in the server current directory.

  • get - Get a file from the server current directory.

  • mget - Get all files that match a wildcard from the server current directory.

It just have to work with anonymous ftp and with only bin mode.

The main Window should have a Help button explaining the functions and operations. It has to have an area to show the directory tree of the server machine. The user should be able to chose files from this area and download them to his machine, and to upload files to the directory represented in this area from a Dialog window. The main window should have a button to open ftp locations.

3.2. WWW Viewer

This program should display HTML pages as links. The user will be able to point the tool to a HTML page and the tool will show all links in that page. The user then can click in one of those links and the tool will:

  • Show the new HTML page's links, if the link is a HTML page (Its name finishes in .html or .htm).

  • Show the figure in another window, if the link points to a .gif or .jpg file.

  • Show a message saying it cannot display that link, if any other kind of link (like Java applets).

The program should have a HELP button that will display a little help and an OPEN button to open URLs.

3.3. SQL Viewer

This program should implement a graphic front for a SQL client, that is able to provide the following functionality

  • Connect to a SQL server that can be in any machine.

  • Allow the user to type a SQL command and submit the command to the server connected.

  • Display the output from the server.

  • Save the output from the server in a file chosen by the user.

  • A history mechanism that allows the user to re-use commands already typed.

The main Window should have a Help button explaining the functions and operations.

3.4. File Manager

The unit FileManager is the only unit for a File Manager Application. The Java File Manager application is designed to provide basic facilities to deal with files in a graphic environment. It allows file system browsing, copying, moving and deleting files and creating folders. It also allows files of a specific type to be associated with an application, and handles opening the file with the proper application when the user requests it.

To show information, the Java File Manager uses the following format:

drw  FileName     FileLength    FileType

where the first three letters indicate wether the file is a directory, is readable and writable. The second column is the name of the file, followed by its size and a description of the File Type. This way you could have:

-rw  FileManager.java   1324    Java Source File

3.4.1. The Menu

Commands are given to Java File Manager mainly through Menus. There are:

3.4.1.1. The File Menu

Open: Opens the currently selected file (or the one nearest the top if more than one are selected) with the proper application. If the selected file is a directory, updates the file list.

Create Folder: Creates a folder with the specified name.

Associate: Collects information about the application to be associated with the selected file. A comment about the file type can also be specified.

Quit: Quit Java File Manager.

3.4.1.2. The Edit Menu

Select All: Selects all files in the current directory.

Cut: Mark selected files to be moved to a new directory.

Copy: Mark selected files to be copied to a new directory.

Paste: Copies or moves the marked files, according to if they were copied or cut.

Delete: Deletes currently selected files.

3.4.1.3. The View Menu

-Order by Name: Orders files by Name

-Order by Type: Order files by type

-Order by length: Order file by length

3.4.1.4. The Help Menu

-Help: Shows this message

3.4.2. The Tool Bar

The buttons Cut, Copy, Paste and Delete shown in the Toolbar are shortcuts for their equivalents in the Edit Menu.

3.5. Chat Viewer

This program should implement a graphic front end for a chat client. It should be able to provide the same functionality as the following commands offer, when connected to the chat server developed in the assignment 2:

        list - list all rooms
        nickname - change the username
        join - join a room
        leave - leave a room

The main window should have a Help button explaining the functions and operations. The window must have three areas: one to show the messages received from the server, one to show the nickname of the users in the active room and one to allow the user to send messages.
