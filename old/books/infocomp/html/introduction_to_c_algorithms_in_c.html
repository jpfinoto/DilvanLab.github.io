3.2. Algorithm Example

Algoritmo Raízes

variáveis
   real a,b,c
   real delta
   real x1,x2

programa
   leia(a,b,c)
   delta= b*b - 4*a*c

   se (delta < 0) então
      escreva('Esta equação não possui raízes reais.')
   senão
      x1= (-1*b - raiz(delta)) / (2*a)
      x2= (-1*b + raiz(delta)) / (2*a)
      escreva('As raízes são ',x1,' e ', x2)
   fim se
fim

3.3. C Program

//Programa Raizes

//variáveis
   float a, b,c;
   float delta;
   float x1, x2;

//programa

int main() {

   printf("Digite o valor de a:");
   scanf("%f", &a);
   printf("Digite o valor de b:");
   scanf("%f", &b);
   printf("Digite o valor de c:");
   scanf("%f", &c);

   delta= b*b - 4*a*c;

   if (delta < 0) {
      printf("Esta equacao nao possui raizes reais.");
   } else {
      x1= (-b - sqrt(delta)) / (2*a);
      x2= (-b + sqrt(delta)) / (2*a);
      printf("As raizes sao %f e %f", x1, x2);
   }
   
}

3.4. Characteristics

  • Semelhança com o pseudo-código;

  • Alguns comandos foram adicionados, mas estruturalmente o algoritmo e o programa são muito parecidos;

3.5. What changes?

  1. A sintaxe torna-se mais rígida, restrita ao vocabulário específico da linguagem de programação;

  2. A pontuação torna-se necessária;

  3. No algoritmo a capacidade de representação (tamanho) e a precisão das variáveis numéricas são infinitos.

  4. Eventualmente alguma construção que fazemos em algoritmos pode não ser eficiente na linguagem de programação (melhorar a eficiência).

  5. Devemos nos preocupar com erros e situações excepcionais, como divisão por zero e variáveis com valores inadequados fornecidos pelo usuário.

  6. Os comentários que identificam o algoritmo e que esclarecem sua construção devem ser transcritos na codificação.

  7. A forma de interação do programa com as pessoas deve ser planejada e incluída no programa.

  8. Os cálculos devem ser codificados de forma a tornálos rápidos e precisos conforme os recursos apresentados pela linguagem.

3.6. Recommendations

  • Procure estabelecer e manter um padrão de documentação.

  • Comente o objetivo do programa, a autoria do programa, os trechos menos óbvios de código, e o significado das variáveis menos óbvias.

  • Escolha mensagens claras e procure definir bons casos de teste para o seu programa.

  • Revise o código desenvolvido pelo menos uma vez para programas pequenos, e pelo menos duas para programas maiores.
