JSML (Java Speech Markup Language)

Trata-se de uma linguagem de marcação derivada da linguagem, também de marcação, Extendend Markup Language (XML). É utilizada para descrever entradas de texto nos sintetizadores da API de programação do IBM Speech Development Kit Java Technology. Seus elementos são capazes de prover informações detalhadas sobre como pronunciar um texto que for submetido ao sintetizador. Estão incluídos elementos que descrevem a estrutura de um documento (parágrafos, sentenças), a pronúncia das palavras e das frases, além de marcadores de texto, elementos de controle de ênfase, ritmo, freqüência, volume, etc. Uma marcação adequada aumenta a qualidade e a naturalidade do texto sendo sintetizado.

O JSML é utilizado somente como argumento para os sintetizadores e pode ser acessado via uma das variações do método speak. Pode ser fornecido um arquivo texto contendo texto formatado em JSML ou podem ser fornecidas pequenas sentenças. Na tabela 4 a seguir podemos ver os tags que constituem o JSML.

Elemento
Descrição
Exemplo
PARA Determina que o texto marcado trata-se de um parágrafo. <PARA>Este é um curto parágrafo.</PARA> <PARA>O assunto mudou, então este é um novo parágrafo. </PARA>
SENT Determina que o texto marcado trata-se de uma sentença. <PARA><SENT>Até, logo. </SENT><SENT>Te vejo logo mais.</SENT></PARA>
SAYAS Especifica como o texto sendo marcado deve ser lido. <SAYAS SUB="0x0">zero a zero</SAYAS>
EMP Determina com qual ênfase o texto será pronunciado. Os valores podem ser: "strong", "moderate", "none", ou "reduced". <EMP LEVEL="reduced"> Cochichar em público é falta de educação.</EMP>
BREAK Determina uma parada durante a síntese. Pode ser definido em milissegundos ou em tamanho: "none", "small", "medium", ou "large". <BREAK MSECS="300"/> ou <BREAK SIZE="small"/>
PROS Determina atributos do discurso: volume (VOL número absoluto entre 0.0 e 1.0), ritmo (RATE em palavras por minuto), freqüência (PITCH em hertz), intervalo de freqüência (RANGE em hertz). <PROS RATE= "50" VOL="+80%">Eu vou falar alto e claro, só mais uma vez. </PROS>
MARKER Quando este elemento é alcançado é gerado um evento markerReached que pode ser interceptado por um SpeakableListener. A tag MARKER pode ser colocada em qualquer local do texto JSML possibilitando monitorar qualquer trecho do texto sendo lido. O parâmetro MARK identifica o marcador. Responda <MARKER MARK= "sim_nao"/> sim ou não.
ENGINE Fornece informações específicas para o sintetizador identificado pelo ENGID, que pronunciará o que está definido pelo parâmetro DATA. Se o sintetizador especificado não estiver disponível, será pronunciado o texto que está marcado. O sapo faz <ENGINE ENGID= "Sapo_Sint1.0" DATA= "&lt;ribbit=1"> sintetizador inadequado </ENGINE>
