Introdução

A tecnologia Speech Development Kit Java Technology é um pacote de programação Java, portanto 100% orientado a objetos, que se utiliza da tecnologia de reconhecimento e síntese de voz comercializada pela IBM com o nome de ViaVoice. A API é uma ferramenta que possibilita a construção de interfaces utilizando-se recursos de voz. Qualquer aplicação standalone Java (não inclui applets) pode ter sua interação usuário-computador estabelecida com este pacote de software.

O reconhecimento e a síntese propriamente ditos não são realizados pelo software Java, reforçando-se que a API é apenas um meio de se construir interfaces orientadas a voz. As aplicações deixam todo o processamento de voz para o software que está por trás, o ViaVoice da IBM (versão comercial ou não).

Assim como os modelos de programação de interface gráfica, o SDK Java também é orientado a eventos. Diferenciando-se porém a natureza destes eventos, que no caso não são desencadeados por I/O como o mouse ou o teclado, mas sim pela voz humana, que chega ao processamento utilizando-se um microfone e a placa de som.

O processamento gerado pela interface também é distinto, sendo capaz de sintetizar voz e estabelecer a completa comunicação verbal com o usuário. Além da síntese de voz, peculiar à tecnologia, qualquer outro tipo de processamento pode ser desencadeado utilizando-se essa técnica de software: desde de evoluções gráficas a até processamento de dados.

Como dito anteriormente, o software baseia-se em duas premissas, a síntese e o reconhecimento de voz:

  • O reconhecimento pode ser realizado com base em gramáticas, que podem ser de duas espécies:

- de ditado: especial para a elocução contínua de discurso, quando o software procura determinar o que está sendo falado através de uma ampla base de dados de palavras e fonemas inter relacionados. A análise da entrada de som é realizada procurando-se estabelecer correspondência entre as palavras pré estabelecidas e o que está sendo dito. Para aumentar o acerto de reconhecimento é feita uma análise automática do contexto em que as palavras estão sendo pronunciadas. Esta base de dados é simplesmente referenciada como DictationGrammar, sendo possível utilizar diferentes gramáticas desta natureza em função do domínio que abrangem: como medicina, engenharia, direito, etc.

- de regras: estas gramáticas utilizam regras, estas regras são definições estabelecidas pelo desenvolvedor para se limitar o conjunto de possíveis eventos sonoros que podem ser disparados pelo usuário. Através de um arquivo texto escrito em um formato especial de definição de gramáticas, o JSGF, determinam-se exatamente todas as possíveis palavras ou frases (regras) que podem ser pronunciadas. A cada uma delas associa-se  um evento específico para ser tratado pelo software. Estas gramáticas são definidas pelo programador e são referenciadas como RuleGrammar.

  • A síntese de voz pode ser realizada através de simples frases fornecidas como argumentos para a premissa "speak". Frases diretas são facilmente sintetizadas mas uma maior naturalidade do sintetizador é conseguida via uma linguagem de marcação denominada JSML, concordante com o padrão XML. Através dela podemos alterar propriedades como o tom de voz, a frequência, ritmo e volume.
