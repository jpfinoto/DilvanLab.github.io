Assigments Format

    The homework assigment should be handed in on a 3.25 diskette (IBM-PC format), or e-mailed to me. It should be devided in three parts:

    • The source files (*.java).
    • The bytecode files (*.class).
    • A HTML file with a UDF (Unit Development Folder).

    All source and text files must contain the names of all students of the group.
    The UDF file should have links to the source files and class files in the code section (You don't have to include the actual code as text).
    If your group decides to e-mail me the files, use the netscape e-mail Attach facility. It includes your file using Mime. This way you can send me any kind of file (including *.class files or pictures).

    Peers Review

    Each group of students will review the UDF of another group. This review must discuss the following topics:

    • The quality of the code generated.
    • The quality of the software design.
    • Could your group have done something better?

    The review should be handed in on a 3.25 diskette (IBM-PC format), or e-mailed to me. It should be a HTML file.

    Grades

    Each group's grade will be based on the quality of its homework assigment (70%) and on its review of the work of another group (30%).

    ASCII files

    You can convert ASCII files to HTML using the following template:

      <HTML>
      <HEAD> <TITLE>          Title of your text        </TITLE>
      </HEAD>
      <BODY BGCOLOR="#FFFFFF">
      <PRE>
                             Your ASCII text
      </PRE>
      </BODY>
      </HTML>

    If you want to include a link to a file or a picture in your file add:

    • For a file: <A HREF= name of the file >
    • For an image (.gif or .jpg): <IMG SRC= name of the image file >

      • Don't forget to send any included file together with your main file.

UDF - Unit Development Folder

    What is a UDF? It is a specific form of development notebook which has proven useful and effective in collecting and orginizing software products as they are produced.

    A UDF is a basic management tool for software development. It is interesting that all students know what a UDF is and how to use one. You can find more information about UDF in the:

      UDF - Unit Development Folder Guide.

    You should adapt the UDF to the needs of your assigment, but try to stick as much as possible to what is described in the guide.
